What is reduce()?

While reduce() is a fairly simple method once you wrap your head around it, I think it is much more complicated to understand than map() or filter(), so this post will be much longer than the respective posts for map() or filter() as a result. According to the Mozilla Developer Network (MDN) web documentation, "The reduce() method executes a reducer function (that you provide) on each element of the array, resulting in a single output value.‚Äù

Now what the heck does that mean for us? reduce() takes an array, iterates through it, and performs whatever actions we instruct it inside of the callback function, returning a new item based upon the actions we provide. Unlike map() and filter(), we have two required parameters inside of the callback function that we provide: an accumulator and a currentValue. The currentValue is the same as map() or filter(): it is the current element in the array that we are performing the iteration upon. However, we also have this weird parameter called accumulator that we have to provide. The accumulator will always be equal to the value we returned in the previous iteration. There is an optional parameter that we can also pass to reduce() (NOT inside of the callback function, as the second parameter in our call of reduce()) called initialValue, that sets the initial value of accumulator. If you do not give an initialValue to reduce() (which as I mentioned, you do not have to), reduce() will simply set the initialValue of the accumulator to the first value in the array which reduce() was called on. Now that was a lot of information thrown at you, and you are probably still confused as to how reduce() works, so let‚Äôs jump into a basic example! 

            
              const ages = [21, 43, 32, 67, 27];

              // Using Math.max() and reduce()
              const maxAge = ages.reduce((accumulator, currentAge) => {
                return Math.max(accumulator, currentAge)
              });
              console.log(maxAge); // 67

In this first example, we are using reduce() to get the maximum value from our array ages. reduce() will loop over our array and for each value in that array, return the higher age between accumulator and currentAge. If you haven‚Äôt seen the Math.max() function before, all it is doing is returning the higher value between whatever two arguments that you pass to it (just like many methods in the built-in Math object, Math.max() is very useful). You'll notice that we don't pass an initialValue to reduce() in this function. While you might initially think that on the first iteration currentAge would be set to 21, it is actually set to 43. ü§Ø What??! Let me explain.

Because we don‚Äôt provide that initialValue to reduce(), our accumulator will be equal to 21 and currentAge will be equal to 43, which means that we will only loop 4 times instead of the 5 that you may have anticipated! If we were to see the values of accumulator and currentAge for each iteration, we would see the following: 

            
              accumulator: 21, currentAge: 43
              accumulator: 43, currentAge: 32
              accumulator: 43, currentAge: 67
              accumulator: 67, currentAge: 27

As you can see, reduce() only loops 4 times since it sets the initial value of accumulator to the first value in our array (21). In each iteration, we return the higher number between the accumulator and the currentAge. Now, you might be asking "Ben. Is this really all that different from running a for loop that uses the same comparison (Math.max(accumulator, currentAge) to change a predefined value?‚Äù No, not really. But once you understand what is going on with reduce(), this syntax not only becomes much more clean to look at for you and other developers you might work with, but unlocks even more powerful code that you can write! This is a fairly trivial example meant to show you in an easy manner how reduce() works. Before we move on to a more complicated example, let‚Äôs refactor the above snippet to demonstrate how to set the initialValue for reduce() as well as use one of my favorite things in Javascript: the ternary operator! 

            
              const ages = [21, 43, 32, 67, 27];

              // Using the Javascript ternary operator
              function getMaxAge(accumulator, currentAge) {
                return accumulator > currentAge ? accumulator : currentAge;
              }
              
              // Call reduce with an initialValue argument this time
              const maxAge = ages.reduce(getMaxAge, 0);
              console.log(maxAge); // 67

If you're not familiar with the ternary operator, it‚Äôs a very useful feature that the Mozilla Developer Network (MDN) will explain in greater detail than I will here. The ternary operator is at its core a shorter syntax for an if/else block of code. You start with an expression the resolves to a truthy or falsy value and use the ? to denote that if that value is truthy, you want to do something based on that expression. Otherwise if the expression is falsy, you will default to some other code that comes after the : .

In the above example, our expression is accumulator > currentAge, the code we want to run if that is truthy is accumulator, and the code we want to run otherwise is currentAge. Notice that we are using this ternary operator statement after a return keyword. That means that if our expression accumulator > currentAge is truthy, we will be returning accumulator. If it is falsy we will return currentAge. While we could use Math.max() to get the higher number between two numbers, I think it more clearly demonstrates what we are checking within reduce(). Every iteration, we are checking whether or not the previous largest value (our accumulator) is greater than the currentAge that we are comparing. If it is, we return the accumulator as we want to continue to remember the greatest value. If currentAge is greater, we return currentAge, which is what reduce() will set the accumulator of the next iteration equal to.

Note that in this example, our reduce() function will run 5 times, since we provide it an initialValue of 0. ‚ÄúBut Ben!" you might say, ‚Äúthis returns 67 just like the first snippet we looked at! Why would we provide an initialValue and make reduce() run an extra time?‚Äù And that would be a great observation! Let‚Äôs look at another example where we can see why explicitly setting an initialValue might be useful to us! 

            
              const people = [
                { firstName: "Ben", lastName: "Smith", age: 21 },
                { firstName: "John", lastName: "Michaelson", age: 43 },
                { firstName: "Wes", lastName: "Tomkins", age: 32 },
                { firstName: "Julie", lastName: "Donaldson", age: 67 },
                { firstName: "Katelyn", lastName: "Roberts", age: 27 },
              ];

              const maxAge = people.reduce((accumulator, person) => {
                return accumulator > person.age ? accumulator : person.age;
              }, 0);
              console.log(maxAge); // 67

In the above snippet, we have an array of objects that contain each person‚Äôs firstName, lastName, and age. If you want to find out who is the oldest person in this array of people, reduce() makes it easy! We simply check in the callback that we provide to reduce() whether or not the accumulator is greater than the age of the current person that we are comparing. If it is, return the accumulator. If it‚Äôs not, return the age of the current person. Here is the value of accumulator and person for each iteration of reduce() from the above example: 

            
              accumulator: 0, person: { firstName: 'Ben', lastName: 'Smith', age: 21 }
              accumulator: 21, person: { firstName: 'John', lastName: 'Michaelson', age: 43 }
              accumulator: 43, person: { firstName: 'Wes', lastName: 'Tomkins', age: 32 }
              accumulator: 43, person: { firstName: 'Julie', lastName: 'Donaldson', age: 67 }
              accumulator: 67, person: { firstName: 'Katelyn', lastName: 'Roberts', age: 27 }

Did you notice that we set an initialValue for our accumulator to 0? Why did we do that? Well let‚Äôs see what happens if we use the same people array and don't include an initial value. Here‚Äôs the code and the result: 

            
              // We aren‚Äôt providing an initialValue this time! 
              const maxAge = people.reduce((accumulator, person) => {
              	return accumulator > person.age ? accumulator : person.age;
              });
              console.log(maxAge); // 67

It returns the same value? Did you spot the bug in this code? Even I missed it at first, but when I played around with this example some more to understand why it returned 67, the answer was clear. This code does not run the way we are wanting it to, even though it does run correctly. On the first iteration, our accumulator is set equal to the first value in people since we don‚Äôt provide an initalValue. That means that for our first iteration, we are asking Javascript to tell us if { firstName: ‚ÄúBen‚Äù, lastName: ‚ÄúSmith‚Äù, age: 21 } > 21. Well clearly we can‚Äôt compare an object with an integer, right? Unlike many other programming languages, Javascript is a bit weird when it comes to what it considers to be true and false. Instead of throwing an error at us when we try to compare an object to an integer (like many other languages would), Javascript evaluates the statement { firstName: ‚ÄúBen‚Äù, lastName: ‚ÄúSmith‚Äù, age: 21 } > 21 as false. So our first iteration returns 43 because when that expression evaluates to false, we return person.age, which is 43. So our code actually continues to execute the same from here on out as when we gave reduce() an initialValue.

What if we were to rearrange the objects in our array? Let‚Äôs move the fourth item in our array to the beginning of our array, and re-run the same code (without an initialValue) and see what happens: 

          
            const people = [
              { firstName: "Julie", lastName: "Donaldson", age: 67 },
              { firstName: "Ben", lastName: "Smith", age: 21 },
              { firstName: "John", lastName: "Michaelson", age: 43 },
              { firstName: "Wes", lastName: "Tomkins", age: 32 },
              { firstName: "Katelyn", lastName: "Roberts", age: 27 },
            ];

            const maxAge = people.reduce((accumulator, person) => {
              return accumulator > person.age ? accumulator : person.age;
            });
            console.log(maxAge); // 43

This returns 43? Now we can start to truly see the problem with not passing an initialValue! If we didn‚Äôt know what this data looked like (for instance if we were pulling data from a database and trying to get the highest age without ever seeing the data ourselves) and we tried to run this, we might not ever notice the issue with this code. But clearly 43 is not the highest age. You and I can both see that clearly! The issue is because our accumulator is equal to the first item in the array by default, and that item happens to have our highest age! In our first iteration, Javascript checks the expression { firstName: ‚ÄúJulie‚Äù, lastName: ‚ÄúDonaldson‚Äù, age: 67 } > 21, which it evaluates to false. That means that person.age, or 21 is passed to the next iteration as our accumulator instead of 67 which is our actual highest age!

This is why passing an initialValue is so useful in certain circumstances. We can set an initialValue of 0 to our accumulator so that on its first iteration, no matter what order our people data is in, it will always be comparing integers in our expression. In the first iteration with our initialValue of 0, it would check if 0 > 21, and we know that for you to be alive you have to be more than 0 years old.
